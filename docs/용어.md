# GEUL & WMS 용어 정리

**버전:** v2.0  
**작성일:** 2026-01-30  
**상태:** 개정판

---

## 1. 최상위 개념

### 1.1 GEUL (General Embedding vector Unified Language)

- 정의:  
  GPT류 LLM과 WMS 사이에서 쓰는 **16-bit 워드 기반 심볼릭 인터페이스 언어**.  
- 특징:
  - 모든 것은 **16-bit Word 배열**로 표현
  - GPT 토큰과 **1:1 매핑 가능**한 정수 시퀀스
  - 의미정렬 식별자 개념을 적용한 **패킷 스트림 구조**
  - **SIDX**를 **TID** 간접 참조로 압축
- 핵심 원칙:
  - **상호 이해성:** 인간과 AI 간 오해 없는 소통
  - **우아한 열화:** 정보 손실 시 상위 개념으로 안전하게 수렴
  - **지식의 화이트박스화:** 모든 정보의 근거(출처, 시점, 신뢰도) 명시

---

### 1.2 SEGLAM (Self-Examination GEUL Architecture Model, 새글앎)

- 정의:  
  GEUL을 사용하여 사유하는 **차세대 AI 아키텍처**.
- 특징:
  - 명확한 역할을 가진 전문 GPT들이 **GEUL Stream**을 통해 협력
  - **의식적 흐름:** 실시간 추론 및 응답 (사용자 요청 → 답변)
  - **무의식적 흐름:** 자기 성찰 및 개선 (WMS 지식/절차 업데이트)
- 핵심 루프:
  - 추론 → 기록 → 성찰 → 개선
- 구성요소:
  - 심상관리 GPT, 쿼리 생성 GPT, 추론 GPT
  - GEUL-Encoder, GEUL-Decoder
  - GEUL-C (캐시), GEUL-Agent

---

### 1.3 WMS (World Management System)

- 정의:  
  여러 세계(World)의 상태를 **그래프 형태로 저장·조회·추론하는 월드 모델 엔진**.  
- 역할:
  - **World** 단위로 FACT / RULE / BOOK 테이블을 관리
  - GEUL 스트림으로 입출력
  - 쿼리, 인덱싱, Rule 평가, SLT 호출 등을 수행하는 **백엔드 엔진**
- 내부 코어:
  - L0 커널 레이어에서 정의된 **ID 체계, SIMD 필터, CSR 인덱스, ETL 파이프라인**을 구현

---

### 1.4 World (월드)

- 정의:  
  하나의 **세계관 / 조직 / 도메인**을 담는 최상위 컨테이너.  
- 예:
  - `World: RealWorld_KR_2025` (현실 세계)
  - `World: MyNovel_Arthur` (소설 세계관)
  - `World: AcmeCorp_ERP` (기업 도메인)
- 구성:
  - 여러 개의 **Table** (FACT / RULE / BOOK)을 가질 수 있음
  - 자체 **타임라인 설정** 보유 (`time_zone`, `calendar`)

---

### 1.5 GEUL-Encoder / GEUL-Decoder

- **GEUL-Encoder:**
  - 정의: 자연어 → GEUL 스트림 변환 모듈
  - 역할: 사용자의 모호한 자연어 입력을 명시적 GEUL 구조로 변환
  - 방식: 95% 심볼릭(패턴 매칭) + 5% LLM(예외 처리) 하이브리드

- **GEUL-Decoder:**
  - 정의: GEUL 스트림 → 자연어 변환 모듈
  - 역할: 구조화된 GEUL 답변을 자연스러운 언어로 번역

---

### 1.6 GEUL-C (글씨, GEUL Cache)

- 정의:  
  사용자 맥락과 심상 정보를 캐싱하는 시스템.
- 역할:
  - 사용자 프로필, 대화 맥락, 가설적 상황 등을 GEUL Stream으로 저장
  - 심상관리 GPT와 연동하여 실시간 업데이트

---

### 1.7 GEUL-Agent (글아)

- 정의:  
  WMS를 살아있는 유기체로 만드는 자율 에이전트 집합.
- 역할:
  - **면역계:** 데이터베이스 건강/일관성 유지
    - 일관성 Agent: 모순 정보 탐지
    - 보정 Agent: 신뢰도 점수 보정
  - **신경계:** 지식 연결/통찰 발견
    - 관계 채굴 Agent: 숨겨진 관계 발견
    - 패턴 발견 Agent: 고차원 패턴 추출

---

## 2. 레이어 / 계층 개념

### 2.0 L0 – Kernel Layer (커널 레이어)

- 역할:  
  **수학·논리·자료구조·성능 최적화**를 담당하는 가장 아래 레이어.  
- 포함 요소:
  - 64-bit SIDX 주소 계산 (Static/Dynamic 구분, Category, Tier, Index)
  - SIMD 비트마스크 필터링, CSR 기반 엣지 인덱스, N-hop 탐색
  - 위키데이터 등 외부 소스를 월드 모델로 가져오는 ETL 파이프라인
- 특징:  
  수학과 논리는 데이터가 아니라 **엔진의 작동 규칙**

---

### 2.1 L1 – World Model Layer (FACT Layer)

- 역할:  
  "지금/그때 그 세계가 **어떻게 되어 있었는가**"를 표현하는 **정제된 상태(canonical state)** 레이어.  
- 단위:
  - CoreTable (FACT 테이블)
  - Timeline Keyframe 기반의 `valid_from` / `valid_until`
- 특징:
  - 잔고, HP, 재고, 관계, 태그, 플래그 등 **상태값**을 표현
  - Canon/Branch 관리(소설 세계관의 분기 등)

---

### 2.2 L2 – Rule Layer (RULE Layer)

- 역할:  
  해당 World를 지배하는 **법칙 / 규칙 / 정책 / 물리 법칙**을 표현하는 레이어.  
- 단위:
  - RuleTable
  - Rule의 유효 기간: `effective_from` / `effective_until`
- 특징:
  - **Rule-as-Data:** Rule도 그래프 데이터로 저장
  - **Rule-as-Code:** Rule Compiler로 컴파일된 형식(Bytecode/IR) 보관
  - Rule 라이프사이클: 정의 → 검증 → 컴파일 → 등록 → 실행 → 폐기

---

### 2.3 L3 – Narrative Layer (BOOK Layer)

- 역할:  
  **서술, 로그, 보고서, 대화, 통화 녹취, 소설 텍스트** 등 "사람이 남긴 기록"을 저장하는 레이어.  
- 단위:
  - BookTable
- 특징:
  - BOOK 자체는 **타임라인 축을 갖지 않음**
  - BookEntry마다 `recorded_at`, `event_time_explicit` 등 메타 기록
  - WORLD 모델과 느슨하게 연결 (`related_entities`, `related_events`)

---

## 3. GEUL 기본 단위 / 구조

### 3.1 Word (워드)

- 정의:  
  GEUL의 최소 단위. **16-bit 부호 없는 정수** (0–65535).  
- 역할:
  - 모든 정보를 Word 단위로 인코딩
  - GPT 토큰과 직접 매핑 가능한 ID 공간

---

### 3.2 SIDX (Semantic-aligned Index, 의미정렬 식별자)

- 정의:  
  **64-bit 의미정렬 식별자** (보통 4 Word로 표현).  
- 역할:
  - "이 개체/개념이 무엇인지"를 전 세계에서 유일하게 식별하는 ID
  - Wikidata QID, 내부 UID 등과 맵핑 가능
- 비트 구조:
  - bit1: Standard(0) / Extension(1)
  - bit2~7: 타입 Prefix (허프만 인코딩)
  - 나머지: 타입별 페이로드
- 영역:
  - **Standard** (`0xxx`): 공식 GEUL 표준
  - **Issuer** (`10xx`): 등록기관 네임스페이스
  - **Proposal** (`1100`): 표준 제안/실험
  - **Free** (`1101`): 완전 자유
  - **Reserved** (`111x`): 차세대 예약
- 참조: `SIDX.md`

---

### 3.3 TID (Temporary ID, 임시 식별자)

- 정의:  
  GEUL 스트림 내부에서만 쓰는 **1 Word(16-bit) 또는 2 Word(32-bit)짜리 임시 ID**.  
- 범위:
  - `0x0000` = NULL / 종결 마커
  - `0x0001 ~ 0xFFFF` = 유효 TID (16비트 기준)
- 역할:
  - 한 스트림 안에서 자주 참조되는 SIDX를 **짧게 압축**해서 쓰기 위한 핸들
  - 자연어의 **대명사**와 유사한 역할
- 특징:
  - TID는 해당 스트림 내에서만 유효 (스코프 제한)
  - 선언된 TID만 참조 가능 (순방향 참조)

---

### 3.4 Packet (패킷)

- 정의:  
  하나의 노드나 엣지를 표현하는 **워드 배열 단위 구조체**.  
- 특징:
  - 고정 길이 헤더 + 가변 길이 페이로드 구조
  - GEUL 스트림 상에서 직렬화/역직렬화에 최적화
- 공통 구조:
  - 1st 워드: Prefix + 타입별 플래그
  - 이후 워드: 타입별 페이로드 + TID

---

### 3.5 Big Endian (바이트 오더)

- 정의:  
  GEUL이 채택한 바이트 오더 표준. **Network Byte Order**와 동일.
- 규칙:
  - 바이트 오더: **Big Endian** (MSB First)
  - 비트 오더: MSB First (bit1 = MSB)
  - 워드 크기: 16비트 (2바이트)
- 선택 근거:
  - SIDX bit1 = MSB 규칙과 일치
  - 사람이 읽는 순서와 동일 (가독성)
  - TCP/IP 표준과 호환
- 참조: `SIDX.md` Section 8

---

## 4. 노드 타입 (Node Types)

### 4.1 Entity Node

- 정의:  
  사람, 장소, 조직, 물건, 추상 개념 등 **"대상"**을 나타내는 노드.  
- Prefix: Standard `0 001` / Proposal `1100 001`
- 모드:
  - **약식 (3워드):** 재참조, 문맥 내 사용
  - **정식 (5워드):** Q-ID 명시, 외부 참조
  - **추상 (3워드):** "모든 학생", "어떤 것" 등 양화 표현
- 속성:
  - EntityType (8비트): 256개 타입 (이진 트리 분류 체계)
  - Attributes (12비트): is_fictional, is_historical, region, era 등
- 참조: `Entity_Node.md`, `개체_상위_분류.md`

---

### 4.2 Quantity Node (물리량/수치 노드)

- 정의:  
  길이, 시간, 질량, 돈, 확률, 타임스탬프 등 **물리량/숫자 값**을 표현하는 노드.  
- Prefix: Standard `0 000 101` / Proposal `1100 000 101`
- 워드: 4~7워드 (가변)
- 특징:
  - 부호(Sign), 값 크기(Size), 값 타입(정수/부동소수점)
  - 단위(Unit Type) + 스케일(Scale) 분리
  - 화폐 확장: 256개 통화 지원
  - 특수 리터럴: TIMESTAMP_SEC, TIMESTAMP_MS, UTF16, RGBA
- 용도:
  - 월드 모델에서 **수치 상태(HP, 잔고, 온도 등)** 정형화
- 참조: `Quantity_Node.md`

---

### 4.3 Meta Node (스트림 제어 노드)

- 정의:  
  GEUL 스트림의 **메타정보**를 표현하는 제어용 노드.  
- Prefix: Standard `0 000 000` / Proposal `1100 000 000`
- 워드: 1~5워드 (타입별)
- Type (4비트):
  - `STREAM_START (0000)`: 스트림 시작, TID 비트폭 선언
  - `STREAM_END (0001)`: 스트림 종료
  - `CREATED_AT (0010)`: 생성 시간 (Unix timestamp)
  - `MODIFIED_AT (0011)`: 수정 시간
  - `CREATOR (0100)`: 생성자
  - `VERSION (0101)`: 버전 정보
- 특징:
  - 모든 GEUL 스트림은 **STREAM_START로 시작**해야 함
  - STREAM_END는 선택 (권장)
- 참조: `Meta_Node.md`

---

### 4.4 Context (맥락) — Node vs Edge

GEUL에서 "Context"는 두 가지로 구분된다:

- **Context (개념):**
  - "이 서술/사실이 어떤 세계/시점/POV/모달리티에서의 것인가?"를 나타내는 추상 개념
  - 예: RealWorld, FictionWorld, 특정 시점, 특정 관점

- **Context Edge:**
  - Context 정보를 다른 Claim(Triple/Verb/Event6 등)에 **연결**하는 Edge
  - "어느 세계관/맥락에서 이 Claim이 참인가"를 명시
  - 상세 정보(출처, 시간, 신뢰도 등)는 별도 Triple로 확장
  - 참조: `Context_Edge.md`

---

### 4.5 Timeline Keyframe Node

- 정의:  
  World의 시간축 상에서 하나의 **앵커 시점**을 나타내는 특수 엔티티.  
- 예:
  - "2025-01-01T00:00:00+09:00"
  - "임진왜란 발발"
  - "MyNovel: Chapter 3 시작"
- 역할:
  - `valid_from / valid_until`, `effective_from / until` 등에서 참조하는 기준점
  - World마다 **전용 FACT 테이블**에 보관

---

## 5. 엣지 타입 (Edge Types)

### 5.1 Verb Edge (동사/서술 엣지)

- 정의:  
  사건/행위를 표현하는 핵심 Edge 타입.
- Prefix:
  - Tiny Verb Edge: Standard `0 1` / Proposal `1100 1`
  - Verb Edge: Standard `0 01` / Proposal `1100 01`
- 모드:
  - **Tiny Verb Edge (2워드):** 고빈도 단순 서술, ~90% 커버
  - **Short Verb Edge (3워드):** 일반 서술
  - **Full Verb Edge (5워드):** 정밀 서술, 완전 한정자
- 동사 한정자:
  - Tense (시제), Aspect (상), Polarity (극성)
  - Voice (태), Mood (서법), Modality (양태)
  - Evidentiality (증거성), Politeness (공손), Confidence (확신도)
- 참조: `Verb_Edge.md`, `동사_상위_분류.md`

---

### 5.2 Triple Edge (속성/관계 엣지)

- 정의:  
  `(Subject, Property, Object)` 형태의 **관계/속성**을 표현하는 Edge.  
- Prefix: Standard `0 000 001` / Proposal `1100 000 001`
- 모드:
  - **기본 모드 (4워드):** PropCode 0~62 (Top 63 속성)
  - **확장 모드 (5워드):** 전체 P-ID 커버
- 의미:
  - "Subject S는 Property P라는 관계로 Object O와 연결된다"
  - 예: `(Alice, FRIEND, Bob)`, `(Apple, headquarter, Cupertino)`
- 참조: `Triple_Edge.md`

---

### 5.3 Event6 Edge (6하원칙 사건 엣지)

- 정의:  
  6하 원칙(Who, What, Whom, When, Where, Why)을 한 번에 담는 **이벤트 표현 엣지**.  
- Prefix: Standard `0 000 011` / Proposal `1100 000 011`
- 워드: 3~8워드 (가변, Presence 비트마스크)
- 구조:
  - Presence 비트 (6비트): 어떤 요소가 포함되었는지
  - Edge TID + 각 요소별 TID
- 요소:
  - **Who:** 행위자 (Agent)
  - **What:** 행위 (Verb Edge TID 참조)
  - **Whom:** 대상 (Patient/Theme/Recipient)
  - **When:** 시간 (Quantity TID 또는 Entity TID)
  - **Where:** 장소 (Location Entity TID)
  - **Why:** 이유 (Clause TID 또는 Entity TID)
- 참조: `Event6_Edge.md`

---

### 5.4 Clause Edge (담화/논리 관계 엣지)

- 정의:  
  서술/사건 간의 **논리적/담화적 관계**를 표현하는 Edge.
- Prefix: Standard `0 000 010` / Proposal `1100 000 010`
- 워드: 4워드
- 관계 타입 (4비트, RST 기반):
  - 인과: CAUSE, RESULT, CONDITION, PURPOSE
  - 시간: SEQUENCE, PARALLEL
  - 대조: CONTRAST, CONCESSION
  - 부연: ELABORATION, BACKGROUND
  - 논증: EVIDENCE, EVALUATION
  - 기타: SOLUTIONHOOD, ALTERNATIVE, MEANS
- 참조: `Clause_Edge.md`

---

### 5.5 Context Edge (맥락 연결 엣지)

- 정의:  
  **"어느 세계관/맥락에서 이 Claim이 참인가"**를 표현하는 Edge.
- Prefix: Standard `0 000 100` / Proposal `1100 000 100`
- 워드: 3워드
- Context Type (6비트, 64개):
  - 출처 (1~20): SYSTEM, USER, NEWS, ACADEMIC, GOVERNMENT, WIKI, API...
  - 파생 (21~30): MODEL, INFERENCE, PREDICTION...
  - 세계관 (31~45): RELIGION, SCIENCE, THEORY...
  - 허구 (46~55): NOVEL, FILM, GAME, LEGEND...
  - 시점 (56~62): NARRATOR, PROTAGONIST, ANTAGONIST, EXPERT...
- 메타데이터 확장: 별도 Triple로 (출처 URL, 신뢰도, 유효 기한 등)
- 참조: `Context_Edge.md`

---

### 5.6 Group Edge (집합/그룹 엣지)

- 정의:  
  복수의 Node를 하나의 그룹으로 묶어 표현하는 Edge.
- Prefix: Standard `0 000 111 000` / Proposal `1100 000 111 000`
- 워드: 4+워드 (가변, 종결 마커)
- GroupType (3비트):
  - `AND (000)`: 논리곱 (conjunction) — "철수와 영희가"
  - `OR (001)`: 논리합 (disjunction) — "A 또는 B"
  - `XOR (010)`: 배타적 선택 — "합격 또는 불합격"
  - `LIST (011)`: 순서 있는 목록 — "1등, 2등, 3등"
  - `SET (100)`: 순서 없는 집합 — "참석자: A, B, C"
  - `RANGE (101)`: 연속 범위 — "1부터 10까지"
  - `PAIR (110)`: 순서쌍 — "(x, y) 좌표"
- 참조: `Group_Edge.md`

---

### 5.7 Faber Edge (코드/AST 엣지)

- 정의:  
  프로그래밍 언어의 **AST(Abstract Syntax Tree)**를 GEUL 그래프로 표현하는 Edge.
- Prefix: Standard `0 000 110` / Proposal `1100 000 110`
- 워드: 4+워드 (가변)
- 명칭 유래: Faber(라틴어) = 장인, 제작자
- 특징:
  - 언어 코드 (6비트): 64개 프로그래밍 언어 지원
  - AST 노드 타입 (8비트): 3+5 분할 (대분류 + 세부타입)
  - 자식 TID: 가변 길이, 종결 마커(0x0000)
- 용도:
  - GoGEUL, PyGEUL 등 코드 분석/변환/생성
  - AI 코드 이해/생성의 중간 표현
- 참조: `Faber_Edge.md`

---

### 5.8 Property Edge (속성 엣지)

- 정의:  
  Entity에 단순 속성값을 부여하는 경량 엣지.
- 특징:
  - Triple Edge의 특수 케이스로 취급 가능
  - Object가 Quantity Node인 경우가 많음

## 6. GEUL 스트림 관련

### 6.1 GEUL Stream (글 스트림)

- 정의:  
  GEUL 패킷들의 **순서화된 시퀀스**. 하나의 완결된 GEUL 문서.
- 구조:
  - `STREAM_START` (필수) → 메타데이터 (선택) → 패킷들 → `STREAM_END` (선택/권장)
- 특징:
  - TID는 스트림 내에서만 유효 (스코프)
  - 순방향 참조만 허용 (선언 후 참조)

### 6.2 TID Scope (TID 유효 범위)

- 정의:  
  TID가 유효한 범위. 기본적으로 **단일 스트림 내**로 제한.
- 규칙:
  - TID는 스트림 시작 시 할당, 스트림 종료 시 해제
  - 스트림 간 TID 참조는 현재 미지원

### 6.3 패킷 순서 규칙

- **선언-참조 순서:** TID 선언은 참조보다 앞에 위치
- **권장 순서:** 
  1. STREAM_START
  2. 메타데이터 (VERSION, CREATED_AT, CREATOR)
  3. Entity Node들
  4. Quantity Node들
  5. Group Edge들
  6. Verb Edge들
  7. Triple Edge들
  8. Event6 Edge들
  9. Clause Edge들
  10. Context Edge들
  11. STREAM_END
- 참조: `Stream_Format.md`

## 11. 기타 세계관 / 철학 관련 용어

### 11.1 Claim vs Fact

- **Claim (주장):**
  - GEUL이 직접 다루는 대상
  - "누군가가 어떤 맥락에서 그렇게 서술했다"는 1차 데이터
  - 상충되는 Claim도 동시 존재 가능

- **Fact (사실):**
  - 여러 Claim을 종합하여 추론한 결과
  - WMS가 L2 Consensus로 계산
  - GEUL은 Fact를 직접 다루지 않음

### 11.2 Canon vs Draft

- **Canon (정전):**
  - 확정된 공식 설정
  - 분기/변경이 제한됨

- **Draft (초안):**
  - 임시/실험적 설정
  - 자유롭게 수정 가능

### 11.3 Modality (모달리티)

- 정의:  
  서술의 확실성/가능성 정도.
- 종류:
  - 현실 (Actual)
  - 가상 (Hypothetical)
  - 가능 (Possible)
  - 필연 (Necessary)

### 11.4 POV (Point of View, 관점)

- 정의:  
  서술의 시점/관점.
- 예:  
  화자 시점, 주인공 시점, 전지적 시점, 빌런 시점
