# GEUL Stream Format

## 1. 🧩 GEUL Stream 구조

* **1st Packet:** GEUL Stream은 반드시 1워드 길이의 메타 헤더로 시작한다.
* **Meta Packets:** 메타 헤더의 옵션에 따라 생성 모델, 생성 시간 등의 메타 패킷이 있을 수 있다.
* **Node Packets:** 가능한 노드 패킷을 전부 나열한 다음
* **Edge Packets:** 엣지 패킷을 나열한다.
* 필요한만큼만 노드 패킷을 나열하고 엣지 패킷을 나열하는 식으로 자유롭게 섞어서 배치할 수 있다. 노드 패킷들과 엣지 패킷들을 완전히 분리하는 것은 권장사항이다.

---

## 2. 📖 Meta Header WORD Packet
* **`GEUL` (1비트):**: GEUL / 사용자 정의
  * `0`: GEUL
  * `1`: 사용자 정의
* **`Ext` (1비트):**:: 확장 여부
  * `0`: 표준
  * `1`: 확장
* **`Length-Index` (2비트):** GEUL Stream '모든 인덱스'의 길이 값
  * `00`: 1WORD 16bit
  * `01`: 2WORD 32bit
  * `10`: 4WORD 64bit
  * `11`: 확장, 별도의 WORD로 길이 지정
* **`Create-Model` (2비트):** 메타 생성 모델 명시 여부
  * `00`: 명시 안함
  * `01`: 2WORD 32bit
  * `10`: 4WORD 64bit
  * `11`: 확장, 별도의 WORD로 길이 지정
* **`Created-Timestamp` (2비트):** 메타 생성 시각 명시 여부
  * `00`: 명시 안함
  * `01`: 2WORD 32bit
  * `10`: 4WORD 64bit
  * `11`: 확장, 별도의 WORD로 길이 지정
* **`Organization` (8비트):** 1비트가 GEUL이 아닌 사용자 정의인 경우, 기관 표시. 0xFF면 확장 워드.

## 3. 🎛️ 패킷

### 패킷의 구조

* **`Node/Edge` (1비트):** 헤더 워드의 첫 비트가 노드/엣지 여부를 결정한다.
  * `0`: 노드
  * `1`: 엣지
* 모든 노드는 적어도 하나의 헤더 워드와 하나의 인덱스 워드로 구성된다.
* 모든 엣지는 페이로드가 인덱스 워드로 구성된다.
* 모든 인덱스 워드의 길이는 메타 헤더 워드 패킷의 `L-Index`가 결정한다.


### 노드

* **`Node/Edge` (1비트):** `0` 노드
* **`Value/Object` (1비트):**
  * `0`: 물리량
  * `1`: 개체


### 물리량

* **`Node/Edge` (1비트):** `0` 노드
* **`Value/Object` (1비트):** `0` 물리량
* **`Unsigned` (1비트):** 부호 여부
    * `0`:signed
    * `1`:unsigned
* **`Value Type` (1비트):** 수의 표현 방식
    * `0`:정수
    * `1`:부동소수점
* **`L-Payload` (2비트):** 페이로드 길이
    * `00`: 1WORD 16bit
    * `01`: 2WORD 32bit
    * `10`: 4WORD 64bit
    * `11`: 확장, 별도의 WORD 1개로 길이 지정
* **`Unit Type` (10비트):** 단위의 종류 1023종. 0x3FF면 확장 워드.


### 개체

* **`Node/Edge` (1비트):** `0` 노드
* **`Value/Object` (1비트):** `1` 개체
* **`Object/Group` (1비트):**
  * `0`: 개체
  * `1`: 그룹
* **`L-Payload` (2비트):** 페이로드 길이
    * `00`: 없음, 대명사를 의미
    * `01`: 2WORD 32bit
    * `10`: 4WORD 64bit
    * `11`: 확장, 별도의 WORD 1개로 길이 지정
* **`Object Type` (11비트):** 명사의 상위 분류 2047종, 0x7FF는 '기타 등등'을 의미


### 엣지

* **`Node/Edge` (1비트):** `1` 엣지
* **`Edge Type` (15비트):** 엣지의 종류 32,767종. 0x7FFF면 확장 워드.
