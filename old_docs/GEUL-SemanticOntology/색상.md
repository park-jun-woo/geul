# 색상 GEUL-SO (Color GEUL-SO)

## 0. 정의

- **색상 패킷**은 색 정보를 표현하는 표준 SEV 패킷.
- 다양한 색상 모델(RGB, CMYK, HSL 등)을 지원하며, 모든 색상 값은 **우아한 열화** 원칙에 따라 부분 손실 시에도 상위 의미로 수렴하도록 설계됨.

---

## 1. 기본 구조

```
[길이 지정자 L][헤더 H: Color][바디 B: 색상 데이터]

```

- **헤더(H)**
    - 의미소 ID에 색상 모델 지정.
    - 예: `0x0C(Color).01(RGB)` / `0x0C(Color).02(CMYK)`
- **바디(B)**
    - 실제 색상 값 (정수/부동소수점).
    - 우아한 열화를 고려해 FP32 → FP16 → INT8로 양자화 가능.

---

## 2. RGB 색상 패킷

- **표현 방식**: `[R, G, B]`
- **범위**: [0–255] (INT8) 또는 [0.0–1.0] (FP32)
- **예시**:
    - 빨강 = `[255, 0, 0]`
    - 회색 = `[128, 128, 128]`
- **우아한 열화**
    - INT8 → “빨강 계열”
    - INT4 → “따뜻한 색”
    - INT1 → “밝음/어두움”

---

## 3. CMYK 색상 패킷

- **표현 방식**: `[C, M, Y, K]`
- **범위**: [0–100] (%) 또는 [0.0–1.0] (FP32)
- **예시**:
    - 순수 파랑 = `[100, 100, 0, 0]`
    - 진한 검정 = `[0, 0, 0, 100]`
- **우아한 열화**
    - K(검정 성분)만 살아 있어도 “밝기 대비” 유지.
    - CMY 일부 손실 → 대략적인 색조(Hue)만 복원.

---

## 4. 색상 패킷의 메타 정보

- **ColorSpace**: RGB / CMYK / 기타(HSL, Lab)
- **Precision**: FP32 / FP16 / INT8 등
- **ConfidenceScore**: 색상이 추정된 것인지, 측정된 것인지

---

## 5. 응용 예시

### 예 1: 이미지 오브젝트

```
Object: 사과 사진
- ColorPacket: RGB(220, 30, 40)  # 사과 표면

```

### 예 2: 인쇄용 데이터

```
Object: 포스터
- ColorPacket: CMYK(0, 95, 100, 0)  # 선명한 빨강

```

### 예 3: 우아한 열화 상황

- 원본: RGB(220, 30, 40)
- 손실(4비트만 남음): “빨강 계열, 중간 밝기”

---

## 6. 설계 철학

- **다중 색상 모델 지원**: RGB는 디스플레이 친화, CMYK는 인쇄 친화.
- **우아한 열화 보장**: 부분 정보만 남아도 “빨강 계열” 같은 상위 개념 유지.
- **의미소 드라이버 확장 가능**: 향후 HSL, Lab, Pantone 등도 드라이버 추가로 지원.
