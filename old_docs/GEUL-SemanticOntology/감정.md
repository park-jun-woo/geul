# 감정 GEUL-SO (Emotion GEUL-SO)

## 0. 정의 및 목적

### 정의
인간과 AI의 감정 상태를 수치화/구조화하여 표현하는 GEUL-SO.

### 목적
- 주관적인 감정을 벡터·확률 분포·범주형 의미소로 정규화
- 대화, 창작, 분석에서 감정적 맥락을 명시적으로 표현
- AI가 감정 표현 ↔ 감정 해석을 정확히 수행할 수 있도록 지원

## 1. 기본 구조

```
[Length L][Header H: Emotion][Body B: EmotionData]
```

- **L (길이 지정자)**: 점진적 모델(HBBB HBBB)
- **H (헤더)**: 감정 패킷 의미소 ID
- **B (바디)**: 감정 데이터 본문

## 2. 감정 표현 방식의 버전

### 2.1 차원형 (Dimensional Model)

#### PAD 모델 (Pleasure–Arousal–Dominance)
- **Pleasure (쾌/불쾌)**: -1.0 ~ +1.0
- **Arousal (활성/비활성)**: -1.0 ~ +1.0
- **Dominance (지배/피지배)**: -1.0 ~ +1.0

#### Circumplex 모델 (Valence–Arousal)
- **Valence (긍정/부정)**
- **Arousal (흥분/차분)**

**예시 (PAD):**
```json
{
  "EmotionPacket": {
    "Model": "PAD",
    "Pleasure": 0.7,
    "Arousal": 0.4,
    "Dominance": 0.2,
    "Confidence": 0.9
  }
}
```

### 2.2 범주형 (Discrete Model)

- **Ekman 6대 감정**: Happiness, Sadness, Anger, Fear, Disgust, Surprise
- **Plutchik의 감정의 바퀴**: 8개 기본 감정 + 강도 단계

**예시 (Ekman):**
```json
{
  "EmotionPacket": {
    "Model": "Ekman6",
    "Emotion": "Sadness",
    "Intensity": 0.8,
    "Confidence": 0.85
  }
}
```

### 2.3 하이브리드형 (Hybrid Model)

**좌표 + 범주 혼합**: 벡터 좌표로 대략적 위치를 잡고, 가까운 범주 감정을 라벨링

**예시**: `[Valence:+0.7, Arousal:+0.5]` → 범주=Joy

### 2.4 문화/언어권별 확장

특정 언어권 전용 감정 단어 지원:

- **일본어**: わびさび (와비사비) → "쓸쓸하지만 고요한 아름다움"
- **독일어**: Schadenfreude → "남의 불행에서 느끼는 기쁨"

*문화적 변형은 심상공간 패킷과 연결하여 관리*

### 2.5 확률 분포 기반

혼합 감정 상태를 표현:

```json
{
  "EmotionPacket": {
    "Model": "Distribution",
    "Emotions": {
      "Fear": 0.5,
      "Anticipation": 0.3,
      "Excitement": 0.2
    },
    "Confidence": 0.92
  }
}
```

## 3. 우아한 열화 (Graceful Degradation)

- **고해상도** → 세부 감정까지 표현
  - 예: "불안 = 공포 0.5 + 긴장 0.3 + 기대 0.2"
  
- **중간 해상도** → 상위 감정군만 표현
  - 예: "부정적 각성 상태"
  
- **저해상도** → 양극적 좌표만 남음
  - 예: "흥분 vs 차분", "긍정 vs 부정"

## 4. RGB–감정 매핑 (색감정 맵)

색상 패킷(RGB, CMYK)과 감정 패킷을 매핑하여 시각–정서적 의미소 브리지 제공:

| 색상 | RGB 값 | 감정 매핑 |
|------|--------|-----------|
| **빨강** | (255,0,0) | {Anger:0.6, Passion:0.3, Danger:0.1} |
| **파랑** | (0,0,255) | {Sadness:0.5, Calm:0.4, Trust:0.1} |

## 5. 메타데이터

- **CreatedBy**: 패킷 생성 주체 (AI/사용자 ID)
- **Timestamp**: DFC 시간 패킷으로 기록
- **Context**: 감정이 발생한 상황(대화/문학/영상)
- **ConfidenceScore**: 감정 해석의 확실성

## 6. 응용 시나리오

1. **대화 이해**: 발화의 감정 뉘앙스를 수치화하여 추론에 반영

2. **창작 지원**: 소설, 시, 영화 제작 시 감정 톤 제어

3. **HCI (Human–Computer Interaction)**: 감정 인식 기반 인터페이스

4. **멀티모달 변환**: 감정 ↔ 색채 ↔ 음악 ↔ 텍스트 간 상호 번역

5. **심상공간 연동**: 특정 세계관/사용자 감정 패턴과 결합

## 7. 요약

- 감정 패킷은 **다양한 버전**(차원형, 범주형, 하이브리드, 문화권별, 분포형)으로 존재 가능
- GEUL의 우아한 열화와 의미소 드라이버 체계 덕분에 병렬 공존과 상호 변환이 가능
- 결과적으로, GEUL는 주관적 감정조차 수치화·벡터화·표준화하여 **AI와 인간이 공유 가능한 언어로 통합**할 수 있음
