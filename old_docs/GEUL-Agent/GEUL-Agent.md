# 글아 (GEUL-Agent): 살아있는 데이터베이스를 위한 자율 지능

## 1. 정의: GDBMS의 무의식적 자아

**`글아(GEUL-Agent)`**는 SEGLAM 아키텍처의 핵심 구성요소로, GDBMS(GEUL Database Management System) 내부에서 24시간 자율적으로 작동하며 지식을 정제하고, 확장하며, 시스템 전체의 건강을 유지하는 **소형 특수 목적 AI 모델들의 총칭**입니다.

인간의 두뇌에 비유하자면, `글아`는 우리가 잠자는 동안 기억을 정리하고 새로운 연결을 만드는 **'무의식'**과 같습니다. 사용자의 직접적인 명령 없이도 배경에서 끊임없이 활동하며 GDBMS를 단순한 데이터 저장소가 아닌, **스스로 학습하고 성장하는 살아있는 유기체**로 만듭니다.



---

## 2. 핵심 철학: 분업화된 지능

`글아` 시스템의 핵심 철학은 **'분업화된 지능(Division of Intelligence)'**입니다. 하나의 거대하고 만능인 AI가 모든 것을 처리하는 대신, 각자 명확한 전문 분야를 가진 수십, 수백 종의 소형 `글아`들이 협력하여 문제를 해결합니다. 이는 다음과 같은 장점을 가집니다.

* **효율성**: 각 `글아`는 자신의 임무에만 최적화된 소형 모델이므로, 거대 모델보다 훨씬 적은 비용으로 빠르고 정확하게 작업을 수행할 수 있습니다.
* **투명성**: 특정 작업에 문제가 발생했을 때, 어떤 전문 `글아`에게 책임이 있는지 명확히 추적하고 수정할 수 있습니다.
* **확장성**: 새로운 기능이 필요할 때마다, 해당 기능에 특화된 새로운 `글아`를 추가하기만 하면 되므로 시스템 전체를 수정할 필요 없이 유연하게 확장할 수 있습니다.

## 3. `글아`의 역할별 분류

`글아`들은 수행하는 역할에 따라 크게 세 가지 범주로 나눌 수 있습니다.

### 3.1 면역계: 데이터베이스의 건강과 위생

GDBMS 데이터의 무결성과 품질을 유지하여 시스템 전체의 신뢰도를 보장합니다.

* **일관성 글아 (Consistency GEUL-A)**: GDBMS 내에서 서로 모순되는 `글오`("지구는 둥글다" vs "지구는 평평하다")를 찾아내고, '모순 원장'에 기록하여 관리자에게 보고합니다.
* **보정 글아 (Calibration GEUL-A)**: 다른 `글아`들이 생성한 정보의 '확신도 점수'가 너무 높거나 낮게 편향되지 않도록, 실제 정확도와 비교하여 점수를 보정하는 '심판' 역할을 합니다.
* **오류 찾기 전문 글아**: `일관성 글아`의 한 형태로, 특정 `글오`들 사이에 존재하는 논리적 오류나 사실 관계의 불일치를 전문적으로 찾아냅니다.

### 3.2 신경계: 지식의 연결과 통찰

기존 데이터를 바탕으로 새로운 지식과 관계를 추론하여 GDBMS를 더욱 풍성하게 만듭니다.

* **추상화 전문가 (패턴 발견 글아)**: 수많은 개별 사례들을 상향식(Bottom-up)으로 분석하여, 그들을 관통하는 일반적인 법칙이나 패턴을 발견하고 새로운 '추상 지식 `글오`'를 생성합니다(예: "모든 사람은 죽는다").
* **일반화 전문가 (관계 채굴 글아)**: 추상화된 법칙을 새로운 사례에 하향식(Top-down)으로 적용하여, 명시적으로 기록되지 않았던 새로운 사실을 추론하고 관계를 연결합니다(예: "에픽테토스는 사람이다 → 에픽테토스는 죽는다").
* **의미소 후보 발굴 글아**: 데이터 속에서 반복적으로 나타나는 새로운 개념을 발견하고, 이를 표준화된 `글소`로 만들 것을 인간 아키텍트에게 제안하는 '언어학자' 역할을 합니다.

### 3.3 실행계: 고차원적 임무 수행

인간이나 다른 시스템의 요청에 따라 복잡하고 창의적인 작업을 수행합니다.

* **사건 추출 전문 글아**: 자연어 뉴스 기사나 보고서를 입력받아, 6하 원칙에 따라 핵심 정보를 추출하고 구조화된 `서술 글오`로 변환하여 GDBMS에 저장합니다.
* **단계별 요약 전문 글아**: 대규모 문서를 의미 단위로 분할하고, 각 부분을 요약한 뒤, 이를 다시 통합하여 계층적인 요약문을 생성하는 분업화된 팀입니다.
* **협력적 창작 전문 글아**: 인간 작가와 협업하여 소설이나 시나리오를 창작합니다. '창작 `글아`'가 아이디어를 내고, '개연성 검토 `글아`'가 설정 오류를 잡아주는 방식으로 작동합니다.

## 4. `글아`의 생성과 추적

`글아`는 인간 개발자가 매번 직접 코딩하지 않습니다. 대신, **범용 대형 모델(GPT 등)이 '설계자' 역할**을 맡아, 자연어 명령을 바탕으로 특정 임무를 수행하는 `글아`의 행동 규칙(`제어 글오`)과 학습 데이터셋을 생성해 줍니다.

또한, 새로 생성된 모든 `글오`의 메타 정보에는 자신을 생성한 **`글아`의 고유 ID가 `생성 주체(CreatedBy)` 필드에 기록**됩니다. 이 덕분에 GDBMS 내 모든 지식의 출처를 완벽하게 추적할 수 있으며, 이는 SEGLAM 아키텍처의 투명성과 책임성을 보장하는 핵심 장치입니다.
