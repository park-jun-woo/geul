# **개체 글오 (Entity GEUL-O)**

## 1\. 정의 (Definition)

\*\*`개체 글오(Entity GEUL-O)`\*\*는 GEUL 언어의 **'명사'** 에 해당하는 가장 기본적인 지식 단위입니다. 세상에 존재하는 고유한 개체, 즉 사람, 장소, 사물, 사건뿐만 아니라 '인공지능'이나 '정의'와 같은 추상적 개념까지도 하나의 `개체 글오`로 표현됩니다.

`개체 글오`는 단순한 데이터 컨테이너를 넘어, 그 자체로 의미를 내포하고 시스템의 고속 검색과 강건성을 보장하는 GEUL 아키텍처의 핵심적인 초석입니다.

## 2\. 핵심 철학

  * **모든 것의 개체화 (Entity-fication of Everything)**: 현실의 인물(`도널드 트럼프`), 허구의 인물(`해리 포터`), 물리적 실체(`지구`), 추상적 개념(`자유`), 심지어 '도착지'와 같은 의미 역할(Semantic Role)까지, 시스템이 다루는 모든 고유 개념은 `개체 글오`로 정의됩니다. 이를 통해 시스템 내 모든 지식 요소가 일관된 형식으로 처리될 수 있습니다.
  * **자기설명적 식별자 (Self-descriptive Identifier)**: `개체 글오`의 가장 혁신적인 특징은, 식별자(ID)가 단순한 주소값이 아니라 데이터의 핵심 정보를 압축하여 담고 있는 **'의미 벡터'** 라는 점입니다. 이는 검색과 추론의 패러다임을 근본적으로 바꿉니다.

## 3\. 절대식별자: 32차원 의미정렬 벡터

`개체 글오`는 위키데이터의 Q-ID와 같은 외부 식별자를 직접 사용하지 않습니다. 대신, 모든 개체는 GEUL 생태계 내에서 고유한 **32차원(64바이트) 의미정렬 벡터형 절대식별자**를 부여받습니다.

  * **설계 원칙**: **'정보 중요도 기반 비트 할당'** 원칙에 따라 설계됩니다. 위키데이터 트리플 통계 분석을 통해, '인간인가?(P31)', '물리적 실체가 있는가?'와 같이 가장 보편적이고 중요한 속성이 식별자의 상위 비트(Most Significant Bits)에 할당됩니다.

  * **궁극의 우아한 열화**: 이 설계 덕분에, `개체 글오`의 모든 상세 정보(`글소` 패킷)가 소실되고 64바이트 식별자만 남는 최악의 상황에서도, 시스템은 상위 비트 정보만으로 **"이것은 적어도 인간이며, 물리적 실체가 있었던 개체이다"** 와 같은 최소한의 핵심 정체성을 파악하고 추론을 이어갈 수 있습니다.

  * **고성능 비트마스크 필터링**: "19세기에 태어난 독일 국적의 남성 철학자"를 찾는 쿼리는 더 이상 여러 테이블을 조인할 필요가 없습니다. GWMS는 1.1억 개의 식별자 리스트를 스캔하며, `is_human=1`, `gender=male`, `nationality=Germany` 등에 해당하는 비트가 모두 켜져 있는 식별자만 찾아내는 단순하고 빠른 비트마스크 연산을 수행합니다.

## 4\. 내부 구조 (GEUL-SO Packets)

`개체 글오`는 프레임(`FE`...`FD`) 안에 여러 `글소` 패킷들을 포함하여 자신의 속성을 상세히 기술합니다.

  * **속성 패킷 (Attribute Packets)**: `개체 글오`의 속성은 **'형용사'** 의 역할을 합니다. '붉은 사과'는 '사과' `개체 글오` 안에 다음과 같은 `색상 글소` 패킷을 포함하는 방식으로 표현됩니다.

      * `[Header: Color_ID, Body: [220, 30, 40]]`

  * **메타데이터 패킷 (Metadata Packets)**: 모든 `개체 글오`는 자신의 출처와 상태를 명확히 하기 위해 표준 메타데이터 패킷을 반드시 포함해야 합니다.

      * `절대ID`, `생성/수정 타임스탬프`, `생성/수정 주체`, `버전`, `확신도` 등.

## 5\. 아키텍처 내에서의 역할과 범위

  * **지식의 근간**: `개체 글오`는 GWMS에 저장되는 지식의 가장 기본적인 단위입니다.
  * **서술의 대상**: `서술 글오(Statement GEUL-O)`는 `개체 글오`들을 주어, 목적어 등으로 참조하여 관계와 사건을 서술합니다.
  * **지식의 원천**: GEUL의 `개체 글오`는 주로 \*\*위키데이터(Wikidata)\*\*의 약 1.1억 개 엔티티를 기반으로 구축되며, 현실과 허구를 총망라하는 방대한 범위를 가집니다.

## 6\. 종합 예시: "지구(Earth)" `개체 글오`

```yaml
ObjectID: [ 0100...1101 ] # 32차원 벡터 식별자
# 상위 비트 해석 예시:
# - bit 0: is_entity = 1
# - bit 1: has_physical_body = 1
# - bit 2: is_human = 0
# - bit 3: is_location = 1
# - bit 4: is_astronomical_object = 1
# ...

GEUL_O_Type: Entity_GEUL-O

Packets:
  - Metadata:
      - AbsoluteID: [ 0100...1101 ] # 자기 자신의 ID
      - ConfidenceScore: 1.0
      - CreatedBy: "Wikidata_Import_Agent_ID"
  
  - Labels: # 이름 정보
      - [Header: Label_ko, Body: "지구"]
      - [Header: Label_en, Body: "Earth"]

  - Attributes: # 속성 (형용사 역할)
      - [Header: InstanceOf_ID, Body: "Planet_ID"] # P31
      - [Header: Mass_ID, Body: [ ... ]] # 물리량 글소
      - [Header: Shape_ID, Body: "Geoid_ID"]
```
