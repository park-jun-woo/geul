# **문서 글오 (Document GEUL-O)**

## 1\. 정의 (Definition)

\*\*`문서 글오(Document GEUL-O)`\*\*는 GEUL 아키텍처에서 최상위 정보 단위를 표현하는 객체입니다. 뉴스 기사 한 편, 연구 논문, 법률 조항, 사용 설명서, 소설, 개인의 생각 묶음 등 하나의 완결된 컨텍스트를 가지는 모든 문서가 `문서 글오`에 해당합니다.

가장 중요한 핵심 원칙은, `문서 글오` 자체는 \*\*의미론적으로 중립적인 순수 컨테이너(Semantically Neutral Container)\*\*라는 점입니다. 문서의 성격과 논리적 구조는 모두 문서 내에 포함된 하위 `글오`들에 의해 정의됩니다.

## 2\. 핵심 철학

  * **의미를 가장 가까운 곳에 (Placing Meaning in the Closest Possible Place)**: `문서 글오`는 자신의 성격(예: 규범적인지, 기술적인지)을 스스로 정의하지 않습니다. 이러한 의미론적 정의는 문서의 내용을 구성하는 개별 \*\*`연결 글오(Connection GEUL-O)`\*\*가 각자 책임집니다. 이 원칙 덕분에 하나의 문서 안에 다양한 성격의 내용이 공존하는 현실 세계의 복잡성을 정확하게 표현할 수 있습니다.

  * **구조적 컨테이너 (Structural Container)**: `문서 글오`의 역할은 문서를 구성하는 여러 `글오`들을 물리적으로 묶어주는 것입니다. 이는 마치 여러 파일을 하나의 폴더나 ZIP 파일로 묶는 것과 같습니다. 폴더 자체의 역할은 파일을 담는 것이지, 그 파일들의 내용을 규정하는 것이 아닌 것과 같은 원리입니다.

## 3\. 필수 구성 글소 (Required GEUL-SO Packets)

`문서 글오`는 일반적으로 다음과 같은 핵심 `글소` 패킷들로 구성됩니다.

| 글소(GEUL-SO) 이름 | 핵심 역할 | 바디(Body) 내용 | 비고 |
| :--- | :--- | :--- | :--- |
| **메타데이터**\<br\>(Metadata) | 문서의 기본 정보를 담습니다. | `절대식별자`, `생성/수정 타임스탬프`, `생성/수정 주체`, `버전`, `최종 커밋 ID` 등 메타 정보 패킷의 집합입니다. | |
| **문서 타입**\<br\>(DocumentType) | 문서의 일반적인 유형을 분류합니다. | `뉴스 기사`, `연구 논문`, `사용자 심상 공간`, `소설` 등 문서의 유형을 가리키는 **벡터 식별자**. | 의미론적 성격이 아닌, 장르나 용도에 따른 분류입니다. |
| **내용 목록**\<br\>(ContentList) | 문서의 본문을 구성하는 핵심 `글오`들의 순서 있는 참조 목록입니다. | `연결 글오`, `서술 글오`, `개체 글오` 등을 가리키는 **벡터 식별자 리스트**. | 이 목록이 문서의 실제 구조와 내용을 결정합니다. |

## 4\. `커밋 글오`와의 관계

`문서 글오`의 버전 관리는 Git의 원리를 따릅니다.

  * `문서 글오`는 버전 관리의 대상이 되는 **'파일'** 입니다.
  * `커밋 글오`는 특정 시점의 변경사항을 기록하는 **'커밋 로그'** 입니다.
  * `커밋 글오`는 `문서 글오` **안에 포함되지 않습니다.** 대신, GWMS의 별도 로그 영역에 존재하며, 변경된 `문서 글오`의 특정 버전을 참조합니다.
  * `문서 글오`의 메타데이터에는 편의를 위해 `최종 커밋 ID` 참조가 포함될 수 있습니다.

## 5\. 응용 예시

### 예시 1: 사용 설명서 (복합적 성격)

```yaml
ObjectID: UserManual_v1.2_ID
GEUL_O_Type: Document_GEUL-O

Packets:
  - Metadata:
      - LastCommitID: "Commit_B_ID"
      - Version: 1.2
  - DocumentType: "사용 설명서(UserManual)_ID"
  - ContentList: [
      "Conn_GEUL-O_Spec_ID",    # 1. 제품 사양 (기술적 연결)
      "Conn_GEUL-O_Rule1_ID"    # 2. 사용법 규칙 (규범적 연결)
    ]
```

### 예시 2: 사용자 심상 공간 (추상적 개념)

```yaml
ObjectID: User_123_Imagination_ID
GEUL_O_Type: Document_GEUL-O

Packets:
  - Metadata:
      - Owner: "User_123_ID"
  - DocumentType: "사용자 심상 공간(ImaginationSpace)_ID"
  - ContentList: [
      "Conn_GEUL-O_Belief_ID",    # "나는 X가 사실이라고 믿는다." (기술적)
      "Conn_GEUL-O_Principle_ID"  # "사람은 Y를 해야 한다." (규범적)
    ]
```
