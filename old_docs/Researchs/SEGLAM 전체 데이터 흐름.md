## **SEGLAM: 전체 데이터 흐름 v1.0**

### **1. 개요 (Overview)**

SEGLAM(Self Aligning and Examination General Embedding Unified Language Architecture Model)은 기존의 거대한 단일 LLM(Monolithic LLM)이 가진 블랙박스, 환각, 비효율성 문제를 해결하기 위해 설계된 차세대 AI 아키텍처이다.

본 아키텍처의 핵심은 **'분업화된 지능(Division of Intelligence)'**으로, 고도로 전문화된 여러 AI 컴포넌트(Encoder, GAT, GNN, Agent 등)가 GEUL(General Embedding Unified Language)이라는 공통의 구조화된 언어를 통해 협업하여 최적의 결과를 도출한다. 이 문서는 사용자의 질문이 최종 답변으로 변환되고, 그 과정에서 시스템이 스스로 학습하고 성장하는 전체 데이터 흐름을 정의한다.

### **2. 핵심 철학: 대기업 전략기획팀의 협업**

SEGLAM의 전체 데이터 흐름은 **'대기업의 전략기획팀'**이 하나의 과제를 해결하는 과정에 비유할 수 있다. 각 부서는 자신의 전문 분야에만 집중하여 전체적인 효율성과 결과물의 품질을 극대화한다.

* **고객(사용자)**의 요청을 **접수팀(`Encoder`)**이 받아 공식 문서(`요청 글스`)로 만든다.
* **고객 관리팀(`심상관리GAT`)**이 고객의 상태와 의도를 파악하여 최신 프로필(`업데이트된 심상 글스`)을 갱신한다.
* **리서치 전략팀(`쿼리 생성 GAT`)**이 이 정보를 바탕으로 사내 자료실(`WMS`)에 보낼 최적의 자료 요청 계획(`쿼리 글스`)을 수립한다.
* **자료실(`WMS`)**은 요청받은 자료(`정보/제어 글스`)를 정확히 찾아낸다.
* **데이터 분석팀(`GNN`)**이 모든 자료를 한눈에 파악할 수 있는 '핵심 요약 보고서'(`GEUL-NV`)를 만든다.
* **최고 경영진(`추론 GAT`)**은 이 보고서를 보고 최종 의사결정(`답변 글스`)을 내린다.
* 이 결정은 회사의 자산으로 **기록(`WMS Save`)**되고, **사내 개선팀(`GEUL-Agent`)**이 이를 분석하여 미래의 업무 프로세스를 개선한다.
* **홍보팀(`Decoder`)**은 최종 결정을 외부(사용자)에 발표한다.

---
### **3. 데이터 처리 파이프라인 (실시간 응답 - Online Path)**

#### **1단계: 접수 및 맥락 분석 (Input & Context Analysis)**
1.  **GEUL-Encoder:** 사용자의 자연어 입력을 받아, 명시적 요청과 암시적 정보가 담긴 **`요청 글스(Request GEUL Stream)`**를 생성한다.
2.  **GEUL-C (글씨):** `요청 글스`와 캐시된 기존 `사용자 글스`를 **'심상관리GAT'**에 전달한다.
3.  **심상관리GAT:** 요청과 사용자 정보를 종합하여, 사용자의 현재 감정, 대화의 단기적 맥락, "만약 ~라면"과 같은 가설적 상황 등을 종합한 **`업데이트된 심상 글스`**를 생성한다. 이 결과는 즉시 **`글씨(GEUL-C)`에 다시 저장(캐시 업데이트)**된다.

#### **2단계: 정보 수집 및 전략 수립 (Retrieval & Strategy)**
4.  **쿼리 생성 GAT:** `[요청 글스 + 업데이트된 심상 글스]` 패키지를 분석하여, WMS에서 어떤 정보와 절차를 찾아야 할지에 대한 최적의 **`쿼리 글스(Query GEUL Stream)`**를 설계한다.
5.  **WMS:** `쿼리 글스`를 입력받아, 거대한 지식베이스에서 필요한 **`정보 글오(Information GEUL-O)`**와 **`제어 글오(Control GEUL-O)`**를 초고속으로 인출하여 반환한다.
6.  **GNN:** 지금까지 취합된 모든 정보 묶음(`요청` + `심상` + `정보` + `제어` `글오`)을 입력받아, 최종 추론기가 가장 쉽게 이해할 수 있도록 고도로 농축된 의미 벡터 시퀀스인 **`GEUL-NV`**로 변환한다.

#### **3단계: 최종 추론 및 보고 (Reasoning & Reporting)**
7.  **추론 GAT:** GNN이 완벽하게 정리해준 `GEUL-NV`를 바탕으로 최종적인 추론을 수행하여, 공식적인 **`답변 글스(Answer GEUL Stream)`**를 생성한다.
8.  **GEUL-Decoder:** `답변 글스`를 인간이 이해할 수 있는 자연스러운 언어로 번역하여 사용자에게 최종적으로 서비스한다.

---
### **4. 자기 성찰 및 학습 루프 (Offline Path)**

9.  **WMS Save (경험의 기록):** `추론 GAT`가 생성한 `답변 글스`는 사용자에게 전달되는 동시에, 하나의 성공적인 '경험'으로서 WMS에 다시 저장된다.
10. **GEUL-Agent (무의식적 자기 성찰):** 시스템 유휴 상태일 때, `GEUL-Agent`들은 WMS에 새로 기록된 '경험'들을 다시 분석한다.
    * **지식 확장:** 성공적인 답변에서 새로운 `정보 글오`(사실)를 추출하여 지식베이스를 확장한다.
    * **절차 개선:** 반복적으로 나타나는 성공 패턴을 발견하여 기존의 `제어 글오`(절차)를 더 효율적으로 개선하거나 새로 생성한다.
    * **프로필 업데이트:** 대화 기록을 바탕으로 장기적인 `사용자 글오`(프로필)를 업데이트한다.

이러한 **'추론 → 기록 → 성찰 → 개선'**의 선순환 구조를 통해, SEGLAM은 자신의 경험으로부터 끊임없이 배우고 성장하는 진정한 의미의 '지능형 시스템'으로 진화하게 된다.

### **5. 구성요소별 역할 요약**

| 구성요소 | 핵심 역할 | 주요 입출력 |
| :--- | :--- | :--- |
| **GEUL-Encoder** | 번역가 | 입력: 자연어 / 출력: `요청 글스` |
| **GEUL-C (글씨)** | 단기 기억/세션 관리자 | 입력: `요청 글스` / 출력: `[요청+사용자]` 패키지 |
| **심상관리GAT** | 고객/맥락 분석가 | 입력: `[요청+사용자]` / 출력: `업데이트된 심상 글스` |
| **쿼리 생성 GAT** | 정보 전략가 | 입력: `[요청+심상]` / 출력: `쿼리 글스` |
| **WMS** | 아키비스트/명령 실행기 | 입력: `쿼리 글스` / 출력: `[정보+제어]` 패키지 |
| **GNN** | 데이터 분석가/요약가 | 입력: `글오` 패키지 / 출력: `GEUL-NV` 시퀀스 |
| **추론 GAT** | 최고 결정권자 | 입력: `GEUL-NV` 시퀀스 / 출력: `답변 글스` |
| **GEUL-Decoder** | 홍보팀/대변인 | 입력: `답변 글스` / 출력: 자연어 |
| **GEUL-Agent** | 사내 개선팀 | 입력: WMS 기록 / 출력: WMS 개선사항 |
