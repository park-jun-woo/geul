# geul → geul-sidx 이관 계획

## 개요

geul 프로젝트에서 SIDX/코드북 관련 파일을 `/mnt/c/Users/mail/git/geul-sidx/`로 이관한다.
기본 원칙: **이동(mv)**, 단 **.log 파일은 복사(cp)** 하여 원본 유지.

---

## 1. entity/ 전체 이관 (41MB, .venv 제외)

**이동:** `geul/entity/` → `geul-sidx/entity/`

포함 항목:
- `entity/CLAUDE.md` — Entity 서브프로젝트 가이드
- `entity/.claude/skills/` — 6개 스킬 (analyst, architect, builder, dba, ontologist, team)
- `entity/references/` — SIDX.md, Entity Node.md, codebooks.json, codebooks_full.json, type_schemas.json, entity_types_64.json 등 15개 파일
- `entity/scripts/` — sidx_encoder*.py, stage1~5*.py, generate_codebooks.py 등 14개
- `entity/plans/` — Phase 4 계획서 5개
- `entity/output/` — 보고서 3개
- `entity/finished/` — Phase 1~3 완료 산출물 (plans, reports, scripts, codebooks)
- `entity/cache/` — db_schema.md
- `entity/templates/` — codebook_verify_prompt.txt

**제외:** `entity/.venv/` (26MB, Python 가상환경 — 새 레포에서 재생성)

**복사 (원본 유지):**
- `entity/finished/output/stage2_run.log`
- `entity/finished/phase2/stage4_run.log`

---

## 2. geulso/wordnet/ 이관 (9.6MB)

**이동:** `geul/geulso/wordnet/` → `geul-sidx/wordnet/`

포함 항목:
- 루트: verbs.json, verbtop*.py, verbtree*.py, wordnet.sql, wordnet_dict.py, ltree.sql, postgres.py, check/debug/count/valid 스크립트들
- `verbtop559/` — sidx559.json, verbclassified559.json, primitive-map.json
- `verbtrees/` — 동사 계층 트리 파일들
- `json/` — 동사 메타 JSON들
- `classified/` — 분류 결과
- `top/` — top 관련 데이터

4. `geulso/wordnet/` 이동
5. `geulso/wikidata/` 이동
6. SIDX/코드북 관련 문서 이동
7. geul 측 빈 디렉토리 정리 (`docs/history/`, `docs/SemanticOntology/` — 비게 되면 삭제)
8. 양쪽 레포에서 `git status`로 확인
