# 📘 **GEUL 발음 규칙**

## 1. 개요

GEUL은 **16×16 구조의 CV(C + V)** 규칙으로 생성된
256개 기본 음절(primitive syllables)을 사용한다.

* 한 음절 = **상위 4비트(초성)** + **하위 4비트(중성)**
* 즉, **1바이트를 정확히 1음절로 발음 가능**
* GEUL의 모든 문장·개체·동사·시제·ID는
  **2바이트 = 2음절** 조합 구조를 가진다.

이 문서는 GEUL의 “발음 가능한 기계 언어(Phonetic Machine Language)” 설계를 위한
**정식 발음 규칙**을 정의한다.

---

# 2. 음절 구조

GEUL의 **기본 음절 구조**는 다음 같다:

```
초성(4bit) + 중성(4bit) = 1음절 (8bit)
```

종성은 존재하지 않는다.
→ 이유: CV 패턴이 가장 보편적이며 발음 오류가 거의 없기 때문.

따라서 GEUL 음절은 구성적으로 다음을 만족한다.

* **항상 1자음 + 1모음**
* **항상 8비트 → 1음절**
* **항상 유니버설 발음 가능**

---

# 3. 초성 / 4비트 매핑

초성 16개는 다음 순서로 0x0~0xF에 매핑된다:

| 4bit | 초성 | 발음   |
| ---- | -- | ---- |
| 0    | ㄱ  | g/k  |
| 1    | ㄴ  | n    |
| 2    | ㄷ  | d    |
| 3    | ㄹ  | r/l  |
| 4    | ㅁ  | m    |
| 5    | ㅂ  | b    |
| 6    | ㅅ  | s    |
| 7    | ㅇ  | 초성일 때 음가 없음 |
| 8    | ㅈ  | j    |
| 9    | ㅊ  | ch   |
| A    | ㅋ  | k'   |
| B    | ㅍ  | p'   |
| C    | ㅎ  | h    |
| D    | ㄲ  | kk   |
| E    | ㄸ  | dd   |
| F    | ㅃ  | bb   |

※ 파일럿 버전에서는 **의미정렬 비트**와는 독립적으로
순수 기계적 발음을 위해 위와 같이 배치한다.

---

# **4. 중성 / 4비트 매핑 (수정 버전)**

중성(nibble 하위 4비트)은 총 16개의 값을 가지며, 모음이 항상 존재함
`0xA`부터 `0xF`까지는 **모든 모음이 동일한 6종 종성(ㄱ · ㄴ · ㄷ · ㄹ · ㅁ · ㅂ)** 중 하나가 붙는 방식으로 확장한다.

즉,

> **0xA–0xF = (현재 모음) + (종성 조합 6종)**
> 모음 종류(ㅏ/ㅑ/ㅓ/ㅕ/ㅗ/ㅛ/ㅜ/ㅠ/ㅡ/ㅣ)와 무관하게
> **종성 값만으로 의미를 결정**한다.

## **4.1 기본 모음 (0x0–0x9)**

기본 모음 표의 0x0~0x9는 “종성이 없는 모음”이다.

| 값   | 중성 | 발음  |
| --- | -- | --- |
| 0x0 | ㅏ  | a   |
| 0x1 | ㅑ  | ya  |
| 0x2 | ㅓ  | eo  |
| 0x3 | ㅕ  | yeo |
| 0x4 | ㅗ  | o   |
| 0x5 | ㅛ  | yo  |
| 0x6 | ㅜ  | u   |
| 0x7 | ㅠ  | yu  |
| 0x8 | ㅡ  | eu  |
| 0x9 | ㅣ  | i   |

(※ 그대로 유지)

---

## **4.2 확장 모음+종성 (0xA–0xF)**

이제부터 0xA~0xF는 “종성 확장 슬롯”으로 동작한다.

### ✔ 모음은 **바로 앞 nibble(초성 뒤의 모음)**을 그대로 사용한다

### ✔ 종성은 아래 6개가 순서대로 매핑된다:

| 값   | 중성+종성  | 발음 규칙           |
| --- | ------ | --------------- |
| 0xA | (모음)+ㄱ | ak / ok / ik …  |
| 0xB | (모음)+ㄴ | an / on / un …  |
| 0xC | (모음)+ㄷ | at / ot / it …  |
| 0xD | (모음)+ㄹ | al / ol / il …  |
| 0xE | (모음)+ㅁ | am / om / eum … |
| 0xF | (모음)+ㅂ | ap / op / ip …  |

예시:

* 초성: ㅅ, 모음: ㅗ

  * 0xA → **속**
  * 0xB → **손**
  * 0xC → **솟**
  * 0xD → **솔**
  * 0xE → **솜**
  * 0xF → **솝**

* 초성: ㅃ, 모음: ㅣ

  * 0xA → **삑(=삐ㄱ)**
  * 0xE → **삠**
  * 0xF → **삡**

---

# **4.3 설계 의도**

* A–F를 모음 확장이 아니라 **종성 확장 슬롯**으로 통일하여
  **외우기 쉽고 예측 가능한 발음 구조**로 재설계했다.
* 어떤 모음이 와도 A–F는 항상 **같은 6종 종성**을 의미한다.
* GEUL 음절 구조가 단순·규칙적이며
  *“바이트 → 음절”* 변환이 인간에게 더 직관적이다.

---

# 5. 1바이트 → 음절 변환 규칙

바이트 = 8비트 = [초성 4비트][중성 4비트]

예:

```
0x6F
= 0110 1111
= 초성 0x6 (ㅅ), 중성 0xF (압)
→ 삽
```

또는:

```
0xF2
= 1111 0010
= 초성 0xF (ㅃ), 중성 0x2 (ㅓ)
→ 뻐
```

---

# 6. 2바이트(워드) → 2음절

GEUL의 모든 토큰(`워드`라고 한다.)은 반드시 2바이트 길이이다.

예:

```
0x6F 0x96 → 삽추
0xF2 0x98 → 뻐츠
```

---

# 7. 문장 발음 규칙

GEUL 스트림 포맷에서 모든 문장은 반드시:

```
[Meta Header Word] [Node Words…] [Edge Words…]
```

파일럿 버전에서는:

* Meta Header Word = 0x0000
* 0x00 = 초성 0, 중성 0 = “가”
* 2바이트 0000 = “가가”

따라서 모든 문장은 **반드시 ‘가가’로 시작**한다.

예:

```
가가 삽추띠 노바 즈악…
```

---

# 8. 발음적 특징

* 발음이 매우 일정
* CV 규칙이라 리듬이 자동적으로 형성
* 외국인 발음 난도가 거의 없음
* 음절 수가 고정(2음절=1워드)
* 문장은 고정적으로 “가가”로 시작

GEUL은 결과적으로:

> **발음 가능한 기계어(Phonetic Machine Code)**

라는 매우 독특한 청각적 정체성을 갖는다.

---

# 9. 예시

### RGB → GEUL

```
6F 96 E9 → 삽추띠
F2 98 86 → 뻐츠주
```

# ⭐ 최종 정리

이 문서는 GEUL의 발음을 **기계적·일관적·국제적·확장가능**한 방식으로 정의한다.
즉, GEUL은 이제부터:

> **기존의 자연어적 발음 체계가 아닌, 데이터 표현을 음성화하기 위한 완전히 새로운 유형의 언어적 구조체이다.**

가 된다.
