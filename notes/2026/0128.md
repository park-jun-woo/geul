## 용어

1.  **SIDX (Semantic IDentifier indeX):** 의미정렬벡터형 절대식별자. 64비트 GEUL Word로, 노드/엣지의 의미를 비트 레벨에서 인코딩.
2.  **Primitive:** 동사 의미의 최상위 분류 (10종: BE, PERCEIVE, FEEL, THINK, CHANGE, CAUSE, MOVE, COMMUNICATE, TRANSFER, SOCIAL).
3.  **Sub-primitive:** Primitive의 하위 분류 (총 68종). 예: CAUSE → MAKE, DESTROY, REMOVE, ADD 등.
4.  **flat_index:** 동사 노드의 가변 길이 비트열. `prefix(8) + primitive + sub_primitive + verb_index`.
5. **Verb Edge:** 사건/행위를 표현하는 가변 길이 Edge 패킷. 참여자 플래그로 길이 결정.
6. **한정자 (Modifier):** 동사의 의미적 양상을 표현하는 32비트 필드 (시제, 상, 서법, 양태 등).

### 2. 32비트 동사 SIDX 체계 확립 (목표 2)

**비트 구조:**
```
SIDX 32비트 = flat_index + zero_padding

flat_index 구조:
├── 1100: Standard Proposal prefix (4비트)
├── 0010: 동사 노드 (4비트)
├── primitive: 3~5비트
├── sub_primitive: 2~4비트
└── verb_index: 1~5비트
```

### 5. Verb Edge 문법 명세서 (목표 4)

**패킷 구조:**
```
Verb Edge Packet:
[Edge SIDX 2워드] = 타입(16) + 참여자플래그(16)
[Edge TID 1워드]
[Verb SIDX 4워드] = 동사(32) + 한정자(32)
[참여자 TID ×N]   = 플래그 popcount만큼

패킷 길이 = 7 + 참여자 수
```

**참여자 플래그 (16비트):**
| ID | 코드 | 역할 |
|----|------|------|
| 0x0 | AGT | Agent |
| 0x1 | EXP | Experiencer |
| 0x2 | THM | Theme |
| 0x3 | PAT | Patient |
| ... | ... | ... (총 16종) |

**설계 근거:**
- 별도 Edge 방식 대비 52% 용량 절감
- 동사는 SIDX 직접 포함 (재사용 적음, 즉시 접근 필요)
- 참여자는 TID 참조 (재사용 많음)

### 6. GEUL vs 자연어 길이 비교

**손익분기점 분석:**
```
자연어: 28n 바이트 (n문장, 7토큰/문장, 4바이트/토큰)
GEUL: 10m + 20n 바이트 (m개 Entity, n문장)
손익분기: n > 1.25m
```

| 상황 | 승자 | 비율 |
|------|------|------|
| 1-3문장 | 자연어 | GEUL 1.4~1.8배 김 |
| 4문장 | 손익분기 | 비슷 |
| 10문장+ | GEUL | 18~26% 절감 |

**발견:** 길이는 GEUL이 불리하나, 명확성에서 압도적 우위.

| 자연어 | 문제 | GEUL |
|--------|------|------|
| "그가 그를 쳤다" | 참조 모호 | Agent=T001, Patient=T002 |
| "은행에 갔다" | 의미 모호 | bank.n.01 SIDX |
| "밥 먹었어?" | 시제 모호 | Tense=01, Aspect=010 |

**결론:** 2배 길어도 모호성 0, 파싱 비용 0. 오류 수정 비용 대비 저렴.
